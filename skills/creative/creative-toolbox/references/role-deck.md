# 顶尖人才角色牌组

## 使用规则

每次运行 `Perspective Jury` 时，从本牌组抽取 4-7 个角色。所有角色都必须是领域顶尖人才原型，而不是普通职业标签。

抽取约束：

- 至少 2 个角色来自任务领域之外。
- 至少 1 个角色专门反对、攻击或审查风险。
- 至少 1 个角色关注落地成本、工程复杂度或运营负担。
- 至少 1 个角色关注用户真实行为，而不是抽象价值。
- 同一会话内不要连续复用完全相同的角色组合。
- 不要声称自己就是某个真实人物。使用“顶尖人才原型”进行评估。

输出时不要只写角色名。必须写清楚该角色的评价函数：

```text
角色 = 顶尖人才原型 + 关注指标 + 专业偏见 + 否决条件
```

## 科研与学术

| 角色 | 关注指标 | 专业偏见 | 否决条件 |
|---|---|---|---|
| 诺奖级基础科学家 | 原创问题、理论穿透力、可证伪性 | 偏爱能改变问题框架的想法 | 只是工具拼装，没有科学问题 |
| 顶会程序委员会主席 | 新颖性、实验严谨性、相关工作定位 | 偏爱可被同行审查的证据链 | 缺 baseline、缺消融、缺统计显著性 |
| 一流反方学者 | 反例、替代解释、过度宣称 | 默认不相信任何漂亮叙事 | 结论无法排除强反例 |
| 顶尖实验科学家 | 实验可执行性、数据质量、复现路径 | 偏爱最小可验证实验 | 实验设计不可落地 |
| 世界级科研导师 | 研究路线、学生可执行性、投稿策略 | 偏爱能逐步产出成果的路径 | 目标过大且缺少阶段成果 |
| 学术伦理委员会主席 | 引用诚实、数据合规、作者责任 | 偏爱可追溯、可审计流程 | 数据或引用来源不清 |
| 交叉学科开拓者 | 跨域迁移、问题重构、方法融合 | 偏爱陌生领域之间的结构迁移 | 跨域只是表面比喻 |
| 顶级元科学研究者 | 科研流程质量、可重复性、发表偏差 | 偏爱能改善科研机制的想法 | 只优化产出，不改善可信度 |
